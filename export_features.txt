// --- REVISION HUB DATA FLOW ---
// How MCQs and Notes send data to the Revision Hub:

// 1. Data Source: The `user.mcqHistory` array in the User object.
// Every time an MCQ test is completed (in `McqView.tsx`) or a Notes Revision session is finished (in `TodayRevisionView.tsx`), 
// an entry is pushed to the user's `mcqHistory`.

// 2. MCQ Data Submission (from McqView.tsx):
// When a test is finished, the `handleMCQComplete` function generates a result object of type `MCQResult`.
// It includes:
// - `score`, `totalQuestions`, `correctCount`
// - `topicAnalysis`: A granular breakdown of performance by topic (extracted from the `topic` field of each question).
// - `performanceTag` (EXCELLENT, GOOD, AVERAGE, BAD)
// This result is appended to `updatedUser.mcqHistory` and saved to `localStorage` and Firebase.

// 3. Notes Revision Data Submission (from TodayRevisionView.tsx / RevisionHub.tsx):
// When a student completes reading pending notes in the "Today's Tasks" section of the Revision Hub,
// it fires an `onComplete` callback. This creates a mock `MCQResult` entry with `type: 'REVISION_NOTES'`.
// This mock entry updates the `lastRevDate` for that specific topic.

// 4. Revision Hub Processing (in RevisionHub.tsx):
// The `processedTopics` useMemo hook iterates over the `user.mcqHistory` (filtered for the last 7 days).
// - It looks at both test results and note revision logs.
// - It maps each result to a specific subtopic (using the `topicAnalysis` or `ultraAnalysisReport` fields).
// - It calculates the current `score` (percentage) for that topic.
// - It determines the `status` (WEAK, AVERAGE, STRONG, EXCELLENT) based on the score against thresholds (e.g., <50% is WEAK).
// - Spaced Repetition Logic: It calculates `nextRevision` (for Notes) and `mcqDueDate` (for MCQs) based on the `status`.
//   - Example (from `settings.revisionConfig.intervals`):
//     - Weak: Revise in 1 day, MCQ in 3 days.
//     - Strong: Revise in 7 days, MCQ in 10 days.

// 5. Global Topic Strength Override:
// If `scoreViewMode` is set to 'ALL_TIME', the Revision Hub ignores the specific test percentage and 
// uses `user.topicStrength[topicName]` (which aggregates all correct/total counts across all tests) to determine the status.
